Gotchas

Note that if that first paragraph is ending up shorter than it should be here is why:

The second paragraph, which has not been truncated yet, is causing a scrollbar to make the window thinner. As soon as the second paragraph is truncated the scrollbar disappears, and the first paragraph is left short.

I don't intend to fix this, because it's a special case, the fix would add overhead, and I want to keep this plugin light because it has to respond fast. However, it's something to be aware of. If it happens to you, one quick fix would be to call truncate twice:

jQuery('p').truncate().truncate();

Another would be to force scrollbars with a little css:

body { overflow-y: scroll; }

Another fix is to trigger a 'resize' event on window (which is a fix I sometimes use to get IE6 to respond to changes it hasn't seen, as a 'resize' makes the window redraw).

And finally, another would be to modify the plugin by inserting, at line 42, another .each() loop that removes the html from each of the elements in the jQuery collection and stores it in their .data('truncate') objects, as the property html. If you then stick o.match into those elements, that would set up the paragraphs to be their ideal target lengths as a starting point.
